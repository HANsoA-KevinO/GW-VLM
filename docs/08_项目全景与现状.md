## 1. 这个项目到底在干嘛(30 秒版)

用**预训练的视觉语言大模型(VLM,Qwen3.6-27B)微调**,做引力波(GW)科学:
- **输入**:LIGO 应变数据画成的 **Q-transform 时频图**(一张图)。
- **输出**:① 有没有引力波信号(**检测** YES/NO);② 源的物理参数(**质量 / 距离 / 自旋**)。
- **核心赌注(论文卖点)**:**基础模型能不能做 GW 科学**——这是 2025 的问题,区别于"造一个更好的 GW 专用网络"(那是物理领域 2020 年代的事)。

## 4. 现在的方案(重做之后)—— 这是当前正在执行的

经过一轮深入讨论,把参数估计和后续实验**整个重新规划**了。详细计划见 docs/07 §7 + 计划文件;这里讲清"为什么这么改"和"现在做到哪"。

### 4.1 参数估计:换形态(为什么 + 怎么换)

**为什么不再用 bin 分类**:
1. bin 分类**信息有损**(连续量压成几档)、**和 LLM 能力错配**(LLM 生成 token 概率 ≠ 物理量后验)、**不是 GW 领域标准**(领域要带不确定度的后验)。
2. **不拿物理领域当 AI 前沿参照**:GW-ML 金标准(匹配滤波、DINGO 归一化流)是优秀物理,但架构是 2020 年代的、不是基础模型范式。

**换成什么:统一基础模型 + 后验头(VLA 式)**——
- 图+应变在 27B 里融合成一个表示,**参数由这个表示上的"后验头"解码**,输出**带不确定度的后验**(高斯:每参数 均值 μ + 标准差 σ),用 NLL 训。
- 检测维持原样(YES/NO 生成式)。
- **为什么这是前沿而非老 ML**:在基础模型的融合表示上接一个 head 出连续量,**正是当下机器人 VLA(RT-2/OpenVLA/π0)的范式**——它们就是用 head 从 VLM 的 hidden 直接出连续动作。"老 ML"是那个**独立的小应变回归网(解耦)**,我们不走那条。
- **经典 1D 网当基线(不是目标)**:用来量"基础模型到底有没有追平/超过经典专家"。三方对照:E2 bins vs 经典网 vs 统一头。

### 4.2 数据:注入管线(为什么是前置)

两轮融合 + E2 都撞同一堵墙:**90 个真实事件太少**,参数/应变学不动。**注入(用 GR 模板波形 IMRPhenom 加进真实 O3 噪声)是唯一能大量造"带已知精确参数 + 可控 SNR"样本的办法**,是 GW-ML 标准配方。
- **E3** = 注入提**检测**弱信号召回;**E4** = 注入喂**参数**(两个都做)。
- **GR 注入用于参数估计名正言顺**(任务标签本就是 GR 参数);你担心的"学不到本质 / 发现不了新物理"是**另一条独立远期线(异常/偏离 GR 检测),本轮不混进来**。
- **关键守卫**:注入只进 train,val/test 全真实(防"合成 vs 真实"的捷径 + 量真实泛化)。

### 4.3 现在做到哪

- ✅ **Part A(参数重构)架构验证通过**:统一后验头代码全建好,3B 上端到端跑通——检测 0.953 没被拖累,后验头产出 NLL/MAE/bin等价/PIT 校准,loss 有限下降。(3B 冒烟数字是噪声、不算数,只证明代码对。)
- ⏳ **A4 经典基线**(快,待做)。
- ⏳ **Part B 注入管线**(大头,待做):`04_inject` 生成波形+注入、`05_render` 渲染图+应变、改 `07` 注入只进 train、装 `.venv-inject`(pycbc/lalsuite)。
- ⏳ **正式 E4**:27B bf16 统一模型在 真实+注入 上训 → 三方对照 + 校准。

## 5. 几个你反复问到的概念(快速回忆卡)

**检测 vs 参数,为什么图擅长检测、应变擅长参数?**
检测只需"有没有那条 chirp 形状",图(时频)够了,甚至有亚感知线索。参数(尤其质量/自旋)藏在波形的**相位**演化里;Q-transform 画成图时把相位丢了 → 图读不出精细参数。应变是**带相位的原始数值** → 参数的正确输入。

**bin 分类 vs 回归 vs 后验?**
- bin 分类:输出一个"档"(离散,信息有损)。
- 回归:输出一个数(点估计,没有"有多确定")。
- **后验**:输出一个**分布**(均值+标准差),= "28.6 ± 2.1"。GW 科学永远要带可信区间,所以后验才对路。

**解耦 vs 统一(参数由谁出)?**
- 解耦:参数交给一个**独立小网络**(= 经典 ML,VLM 不参与)。
- 统一:参数从 **VLM 的融合表示**上接 head 出(= VLA 前沿范式)。**我们选统一,小网当基线。**

**后验头怎么工作(简版)?**
取 VLM 在"最后一个提示 token"处的隐藏向量(它已看过全部图+应变)→ 过一个小 MLP → 出每个参数的 (μ, logσ) → 用 NLL 训。评估时反演回物理量 + 看 PIT 校准(50%/90% 可信区间命中率该≈0.5/0.9)。

**数据墙 + 为什么注入是前置?**
GW 界训应变模型用**数万条注入**;我们只有 90 事件。参数/融合的价值被"够不够数据"门控住 → 不先造数据(注入),参数这关绕不开。

**thinking 为什么全程关?**
E2 诊断:开 thinking 会让模型用"找明亮 chirp 脊"的弱启发式覆盖学到的亚感知特征 → 把弱信号判 NO。`enable_thinking=False` 不是设限,是还原训练分布。

## 6. 工程现状 / 踩过的坑(快速回忆)

- **跑在哪**:DGX Spark(`super-cortant`),Qwen3.6-27B。检测/参数训练在 Spark,数据管线在 Mac 本地。
- **27B 能跑起来的关键**:它是混合**线性注意力**架构(qwen3_5),默认走慢回退(~20s/batch);本地编译 **fla + causal_conv1d**(Blackwell/CUDA13 能编)激活快路径 → ~5–6s/batch。**bf16 + 梯度检查点**(裸 bf16 前向要 117G 会 OOM,梯度检查点压到 ~79G)。
- **Gemma4 为什么没用**:它是 PLE(逐层嵌入)架构,和 inputs_embeds 注入式融合不兼容(只给 embeds 会反推 input_ids → 爆显存)。
- **代理/链路教训**:Mac IP 会漂(代理地址跟着变);SSH 在 27B 重负载下会抖/断 → 用 `setsid` detach 跑、轮询日志。
- **刚修的关键 bug**:负样本参数 target 是 NaN,在 NLL **结果**上消毒太晚——反向链 `0×NaN=NaN` 污染头梯度 → 全 nan。修法:进计算**前**就把 NaN 消毒。用小 3B 快速隔离掉的。
