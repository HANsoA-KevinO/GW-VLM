# E2(多任务:检测 + 物理参数)结果与思考诊断

> 日期:2026-06。模型:Qwen3.6-27B(E1 最佳)+ 获胜配方(r32/α32/lr2e-4/dropout0/有效batch8)+ viridis 图,3 epoch(30.2h)。
> 数据:`output/training_data/e2`(多任务 schema:`{detection, chirp_mass_bin, distance_bin, chi_eff_bin}`),2394/306/270。
> 评估脚本:`evaluation/evaluate_e2.py`;结果与逐样本归档于 `output/runs/e2_qwen36_27b_viridis/`。

---

## 1. 一句话结论

**多任务(加物理参数)没有拖累检测、反而略升(ROC-AUC 0.940→0.959);参数里只有"距离"明显可学(响度可读),质量勉强、自旋读不出。** 另外一个重要的方法学发现:**开启模型"思考(thinking)"会把检测打崩(把几乎所有样本判 NO)——因为检测靠的是亚感知特征,"讲道理"会用一个更弱的显式启发式覆盖它。**

## 3. 关键方法学发现:为什么"思考"会毁掉检测

评估生成参数时发现:**Qwen3.6 默认 generation prompt 以 `<think>` 开头,模型先输出推理独白,而它是用纯 JSON(无思考)SFT 的**。带思考自由生成时,**270 个样本全被判 NO(生成式 recall=0)**;而 teacher-forced(不思考、直接读 logit)是 ROC-AUC 0.959。

读思考原文(`output/runs/e2_qwen36_27b_viridis/think_diag.jsonl`,60 条)看到失败机制:
- 模型套用**教科书式强信号先验**——"真信号 = 一条明亮的黄色 chirp 脊,Q 值 >5–10";
- 我们的真实 GW chirp 在 viridis Q-transform 里**大多不是肉眼可见的亮线**(弱信号视觉上看不见);
- 于是思考显式推理出"没有清晰亮脊 → NO",**把不思考时能用的亚感知细微证据"推理掉了"**(50 条里只有 2 条因看到罕见亮线判 YES)。

**结论与影响:**
1. **E2 评估必须 `enable_thinking=False`**(还原训练分布,用学到的感知而非把它推理掉)——这不是给模型设限。
2. **对"要不要给检测加 CoT/推理式训练"**:这是强反对证据(与调研里"CoT 伤感知任务"一致)。
3. **深层洞见**:认真看图推理都看不出信号、而不思考的模型却能检出(留出集 AUC 0.96、按事件切无泄漏)——说明检测器靠的是**更底层、非视觉显性的模式**,而非"看懂 chirp"。这对论文叙事和后续方法设计(目标函数对齐 / 多模态物理融合)都是关键依据。

## 4. 工程注记

- **慢的根因**:27B bf16 单流解码是 DGX Spark(GB10)统一内存带宽(~273 GB/s)瓶颈,每 token 要读完整 ~54GB 权重 → 实测 **~3.1 tok/s**,数据中心 H100(~3TB/s)能快十几倍。
- **评估须用原生 transformers + PEFT(`--no-unsloth`)**,绕开 Qwen3.6 在 Unsloth 下的 `get_rope_index` rope bug(见 `docs/04` §2.5)。
- 产物归档:`output/runs/e2_qwen36_27b_viridis/`(adapter + 3 checkpoint + e2_report.json + e2_per_sample.jsonl + think_diag.jsonl + logs/)。
